### 萬能節目結構模板

無論主題類型，成功的 Podcast 節目遵循一個**三層十段**的框架結構：

```
第一層：開場（佔總時長 5-10%）
  ① 冷開場 / 精彩片段預覽（10-20 秒）
  ② 品牌 Jingle / 開場音樂（5-10 秒）
  ③ 主持人自我介紹 + 本集價值承諾（15-30 秒）

第二層：主體內容（佔總時長 70-80%）
  ④ 主題引入與背景鋪陳（1-2 分鐘）
  ⑤ 核心內容段落一（含數據/故事/論點）
  ⑥ 轉場 + 核心內容段落二
  ⑦ 互動/廣告中插（1-2 分鐘，選用）

第三層：收尾（佔總時長 10-15%）
  ⑧ 重點摘要（1-2 分鐘）
  ⑨ 行動呼籲 CTA（15-30 秒）
  ⑩ 下集預告 + 片尾音樂
```

不同節目類型的結構變體：

- **訪談型**：開場 → 來賓簡介（口語化，不念 LinkedIn）→ 3-5 個預設問題（留空間自然延伸）→ 錨點問題（扣回核心主題）→ 快問快答 → CTA
- **獨白型**：強力 Hook → 主題設定 → 清晰分章的談話要點（每段：論點 → 證據 → 故事 → 啟示）→ 總結 → CTA
- **故事敘事型**：第一幕（場景/人物/事件引爆點）→ 第二幕（升高衝突/嘗試與失敗）→ [廣告] → 第三幕（高潮/轉折/結局）→ 反思 → 下集預告

### 起承轉合：中文敘事的獨特優勢

起承轉合是源自中國詩歌的四段式結構，與西方三幕劇（依賴衝突/解決）不同，它依賴**綜合與和諧**來揭示多面真相：

- **起（Introduction）**：建立場景、人物、情境——回答「我們在談什麼」
- **承（Development）**：深化細節、堆疊資訊——不做大變化，持續鋪陳
- **轉（Twist）**：出現一個看似無關的轉折——打破預期，引發好奇
- **合（Conclusion）**：所有元素匯流，揭示更深層的真相——不一定完全解決，但四個部分結合展現更深的理解

這個結構特別適合探索複雜議題的節目——目標不是簡單的問題解決，而是展現 nuanced 的理解。在宮崎駿電影和《寄生上流》中都能看到這種結構的應用。

### 開場白（Hook）的七種武器

你只有大約 **7 秒**建立第一印象，前 **30-60 秒**決定聽眾去留。超過 90 秒的開場，流失率幾乎**翻倍**。以下是經過驗證的七種 Hook 手法：

1. **冷開場蒙太奇**：截取節目中最精彩的 10-20 秒片段前置播放
2. **挑釁性提問**：「如果你對 ______ 的所有認知都是錯的呢？」
3. **驚人數據**：「你知道嗎？89% 的 ______ 其實 ______」
4. **大膽反論**：陳述普遍接受的觀點，然後立即挑戰它
5. **故事鉤子**：簡短個人軼事，設定本集主題
6. **價值承諾**：「聽完這集，你會確切知道如何 ______」
7. **社會證明**：提及聽眾成功案例或近期評論

### 聽眾留存的全程策略

**前 5 分鐘防流失（最關鍵）**：Podcast 在前 5 分鐘會流失 **20-35%** 的聽眾。策略包括——跳過冗長的寒暄（「你這週過得怎樣」對新聽眾毫無價值）、以精彩片段冷開場、立即陳述價值主張、用能量和自信開場、品牌標識壓縮到 5-10 秒。

**中段保持專注力**：交替快節奏與慢反思段落、在廣告/中插前製造懸念（「但接下來發生的事改變了一切⋯」）、穿插迷你故事而非一大段論述、向聽眾拋出思考問題、善用音效和背景音樂做段落切換。

**結尾 CTA 設計**：每個位置只放**一個** CTA（多個請求會壓倒聽眾）、控制在 **15-30 秒**、必須與本集內容相關（不要通用的「記得訂閱」）、用「三明治技巧」——節目前段預埋 CTA 伏筆、結尾正式遞送。

**跨集留存（讓聽眾持續追蹤）**：固定發布時間建立聽眾習慣、維持穩定的節目時長範圍、每集結尾預告下集（製造期待）、系列化內容設計（多集連續主題）、將聽眾導入電子報/社群（從平台聽眾轉為自有受眾）、製作「從這裡開始聽」的入門集供新聽眾使用。

### Show Notes 撰寫要點

優質 Show Notes 同時服務 SEO 和聽眾體驗：

- **吸引人的摘要段落**（50-150 字）——不要洩露所有內容，誘使點擊收聽
- **時間戳記**——每個關鍵段落的時間碼（每個都是 SEO 關鍵字機會）
- **核心要點**——3-5 個主要洞察的條列清單
- **來賓簡介 + 連結**
- **節目中提及的所有資源連結**
- **單一明確的 CTA**
- **逐字稿**（大幅提升 SEO 和可及性）

有播客主實測，在優化 Show Notes 的 SEO 技巧後，搜尋能見度在 20 天內提升了 **53%**。
